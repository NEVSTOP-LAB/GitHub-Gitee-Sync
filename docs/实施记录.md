# 实施记录

本文档记录 GitHub-Gitee-Sync 工具的实际实现结构与关键技术选择，以及实施过程中根据代码审查反馈所做的改进。

> 相关设计规划文档请参见 [docs/计划/](计划/) 目录，技术调研请参见 [docs/调研/](调研/) 目录。

## 模块结构

项目从初始的单文件 `sync.py` 重构为多模块结构，各模块职责如下：

```
├── sync.py              # 主入口: 参数解析 (parse_args)、主同步编排 (sync_all)
├── lib/
│   ├── __init__.py
│   ├── utils.py         # 通用工具: 日志配置、HTTP 请求封装与重试、Token 脱敏、GIT_ASKPASS 认证
│   ├── github_api.py    # GitHub REST API 封装: Token 验证、仓库列表/创建/详情/元信息
│   ├── gitee_api.py     # Gitee REST API 封装: Token 验证、仓库列表/创建/详情/元信息
│   └── sync_repo.py     # 单仓库同步: mirror_sync、sync_repo_metadata、sync_extras
│                        #   extras: releases、wiki、labels、milestones、issues
├── entrypoint.sh        # Docker/Action 入口: 将 INPUT_* 变量映射为标准环境变量后调用 sync.py
├── action.yml           # GitHub Action 元数据: inputs/outputs/branding
├── Dockerfile           # Docker 容器定义 (python:3.11-slim + git)
├── requirements.txt     # 生产依赖 (requests)
├── requirements-dev.txt # 开发依赖 (pytest, pytest-cov)
├── pytest.ini           # 测试配置
└── tests/               # 单元测试 (144 个测试用例)
    ├── test_utils.py
    ├── test_github_api.py
    ├── test_gitee_api.py
    ├── test_sync_repo.py
    └── test_sync.py
```

## 关键设计决策

| 决策项 | 选择 | 原因 |
|-------|------|------|
| HTTP 库 | `requests` | 简洁易用，社区广泛，唯一外部依赖 |
| Git 操作 | `subprocess` | 无额外依赖，可靠稳定 |
| 同步方式 | `git clone --mirror` + `git push --mirror` | 完整同步所有分支和标签 |
| Git 认证方式 | GIT_ASKPASS 临时脚本 | Token 不出现在 URL/进程列表中，避免泄露 |
| Docker 基础镜像 | `python:3.11-slim` | 体积适中，兼容性好 |
| Action 类型 | Docker Container Action | 复用已有 Dockerfile，完整环境控制 |
| Action 入口 | `entrypoint.sh` 桥接脚本 | 映射 `INPUT_*` 变量，兼容直接运行和 Action 两种模式 |
| 仓库创建 | 可配置 `create_missing_repos` | 默认自动创建，可关闭仅同步已有仓库 |
| Release 资产下载 | asset URL + `Accept: application/octet-stream` + Authorization | 支持私有仓库 asset |
| Release 资产写入 | 流式写入临时文件 | 避免大文件 OOM |

> 完整开发步骤请参见 [docs/计划/开发步骤.md](计划/开发步骤.md)。

## 代码审查反馈实施记录

以下列出根据 PR review 改进的实现细节（参见 PR #2）：

### 安全改进
- **Token 不再内联到 URL**：`build_clone_url()` 返回无凭据 URL，`make_git_env()` 通过 GIT_ASKPASS 临时脚本传递 Token，Token 经 base64 编码避免 shell 注入。

### 鲁棒性改进
- **Rate Limit 防御性解析**：`X-RateLimit-Remaining` / `X-RateLimit-Reset` 头使用 `try/except` 解析，缺失或非数字时回退到安全默认值，不会因代理/CDN 异常响应而崩溃。
- **5xx 瞬态错误重试**：HTTP 502/503/504 视为可重试错误，加入指数退避逻辑。
- **Release 资产文件描述符泄漏修复**：改用 `tempfile.mkstemp()` + `os.fdopen()` context manager，保证所有代码路径均关闭文件句柄。
- **流式响应连接泄漏修复**：`requests.get(..., stream=True)` 改用 `with` 语句，确保连接在 `continue` 分支时也被关闭。
- **分页请求非 200 响应警告**：`paginated_get()` 在非 200 时记录警告（含状态码和响应摘要），不再静默中断。

### 同步正确性改进
- **Owner 二次验证**：`/user/repos` 返回的仓库按 `owner.login` / `namespace.path` 过滤，避免 Token 与 Owner 不一致时同步错误账号。
- **`include_private` 应用于 Gitee**：`get_gitee_repos()` 增加 `include_private` 参数。
- **Release 更新路径**：已存在的 Release 检测元信息差异后调用 PATCH 更新，并补充目标端缺失的 assets。
- **Label name URL 编码**：PATCH 请求中 label name 使用 `urllib.parse.quote()` 编码，支持含空格等特殊字符的标签名。
- **Label description 始终发送**：即使 description 为空字符串也发送，允许清空目标端的标签描述。

### entrypoint.sh 改进
- 使用 `exec python /app/sync.py` 替代 `python /app/sync.py`，使 Python 进程成为 PID 1，信号处理和退出码更可靠。
